Healthwise Clinic

Web Development Responsive HTML5 CSS3 Bootstrap JavaScript PHP jQuery SEO

The Healthwise Clinic website was more of a weekend challenge rather than a fully planned project. The challenge was to design and code a responsive website over a weekend from scratch. I gladly took on the challenge, and this is what I came up with.

For the most part, healthwiseclinic.ca is a simple website with heavy focus on front-end. And for that (the front-end), I decided to go with Bootstrap for the main design, since my friend needed the website to be responsive. I kept the tinkering in jQuery to a minimum since I had a time limit, so I just used the default bootstrap arsenal. I wrote some simple JavaScript validation to reduce the stress on my main validation code (written in PHP on the back-end) for requesting appointments. I have also put in some simple SEO optimizations (link to Google+, Facebook description, etc.) just to make things easier for the business owner.

Overall, it was a fun challenge which I enjoyed. I wouldn't change much if I didn't have the time limit, except for cleaning up the markup to make it more W3C compliant.

MedicSpa

Web Development Responsive HTML5 CSS3 Bootstrap JavaScript jQuery jQuery easing SEO W3C Compliance

MedicSpa was an interesting project I worked on for about 2 weeks. I followed standard procedure for the whole project: Sat down with the client, collected requirements, discussed needs, brainstormed for UI/UX designs and color themes. Once all of that was done, I had a clear idea on what needs to be done, and I did it.

medicspa.ca looks like a simple front-end website, and for the most part, it is. I went with an elegant, bright, and clean design (which is what the customer wanted), built on Bootstrap to ensure responsivness. I wrote my own custom CSS, and my own custom JavaScript/jQuery to load and shuffle images. I have also put in some simple SEO optimizations (link to Google+, Facebook description, etc.) just to make things easier for the client.

I am mostly proud of the custom scripts I wrote for the project, and of course, the fact that the website is written in W3C compliant HTML5.

Osgoode Mobile

iOS 9 Swift 2 XCode Sketch3 Ubuntu Server Node.js Parse Server MongoDB Agile Apple's App Store MarkDown Technical Documentation Lead Developer

This is by far the biggest challenge I had. I was in charge of the whole project, so I had to lead 2 junior developers (later became only one) to get everything about this app done. Did I mention that I had no clue about app development (neither iOS, nor Swift) before? Yeah, it was fun.

As the Lead Developer for this project, I was in charge of everything regarding the app. I did the initial designs and requirement gathering, I wrote the technical plan, I planned both the front and back ends of the app, and presonally coded everything on the frontend. The project was using a daily SCRUM Agile method, and the team was using a private GitLab instance (which I personally deployed, as a previous project) for version control.

I coded & tested everything, from the ground up, for the frontend (the iOS app). I used XCode 7 and Swift 2 to write the code, and used various APIs and SDKs: CoreData, GCD, UIKit, Parse iOS SDK, and few more.

Thankfully, I had some help for the backend from my junior developer, so I did the technical design, and he did the majority of the coding and implementations. I had to write few custom Node.js modules only. The backend is built on a Ubuntu Server machine, running Parse Server with an Apache2 instance as reverse-proxy, with few custom Node.js modules, and MongoDB.

Why is this project special to me? It was my first fully managed project. I was in charge of everything, and I made a product that aligns with the high standards set by Osgoode Hall Law School and York University.
