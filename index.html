A bit about me

I'm a technology enthusiast and a world-class nerd set on a mission to make the world a better place!

  • When did you start programming?
  • I had officially written my first line of code (in Basic) when I was 4 years old.
  • Of course, because you're a genius ...
  • No, it's just because I was lucky enough to have an amazing family of geeks where everyone is a Computer major.
  • Programming is a bit too broad, any favorite topics?
  • Yep! While I nerd-out over everything technology related, the fields I am most interested in are Mobile, Web, and Machine Learning.
  • What's your definition of "Programming"?
  • It's the resulting art from the combination of: hardwork & practice, curiousity, and problem-solving.
  • Why do you think you would make a good programmer?
  • I am not (nor do I aim to be) a good programmer. "Good" is a useless subjective target. I strive to be a "Better" programmer.
  • Ok, ok. If we recognize that you're a "better", talented programmer, would you rather make a lot of money, or help others?
  • I would rather use my talent(s) to help others while making enough money to have a roof over the heads of the ones I love.

Healthwise Clinic

Web Development Responsive HTML5 CSS3 Bootstrap JavaScript PHP jQuery SEO

The Healthwise Clinic website was more of a weekend challenge rather than a fully planned project. The challenge was to design and code a responsive website over a weekend from scratch. I gladly took on the challenge, and this is what I came up with.

For the most part, healthwiseclinic.ca is a simple website with heavy focus on front-end. And for that (the front-end), I decided to go with Bootstrap for the main design, since my friend needed the website to be responsive. I kept the tinkering in jQuery to a minimum since I had a time limit, so I just used the default bootstrap arsenal. I wrote some simple JavaScript validation to reduce the stress on my main validation code (written in PHP on the back-end) for requesting appointments. I have also put in some simple SEO optimizations (link to Google+, Facebook description, etc.) just to make things easier for the business owner.

Overall, it was a fun challenge which I enjoyed. I wouldn't change much if I didn't have the time limit, except for cleaning up the markup to make it more W3C compliant.

Osgoode Mobile

iOS 9 Swift 2 XCode Sketch3 Ubuntu Server Node.js Parse Server MongoDB Agile Apple's App Store MarkDown Technical Documentation Lead Developer

I was in charge of the whole project, so I had to lead 2 junior developers (later became only one) to get everything about this app done.

As the Lead Developer for this project, I was in charge of everything regarding the app. I did the initial designs and requirement gathering, I wrote the technical plan, I planned both the front and back ends, and presonally coded everything on the frontend. The project was using a daily SCRUM Agile method, and the team was using a private GitLab instance (which I personally built, on a previous project) for version control.

I used XCode 7 and Swift 2 to write the code, and used various APIs and SDKs: CoreData, GCD, UIKit, Parse iOS SDK, and few more.

The rest of the team worked on coding the backend which I designed. I had to write few custom Node.js modules only. The backend is built on a Ubuntu Server, running Parse Server with an Apache2 instance as a reverse-proxy, with few custom Node.js modules, and MongoDB.

Why is this project special to me? It was my first fully managed project. I was in charge of everything, and I made a product that aligns with the high standards set by Osgoode Hall Law School and York University.
